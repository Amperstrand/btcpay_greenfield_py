# Migration guide: v0.x (2023 urllib3) → v1.x (2025 httpx)

The v1.0.0 release is a **full rewrite**. The old client was auto-generated by
`openapi-generator-cli` using the `python` template (urllib3, sync-only,
class-per-resource). The new client is auto-generated by `openapi-python-client`
(httpx, sync + async, function-per-endpoint, attrs models).

There is **no backwards compatibility**. This guide walks through the common
migration patterns.

## Installation

```bash
# Old
pip install btcpay_greenfield_py  # (was never published to PyPI)

# New
pip install btcpay-greenfield-py
```

## Client construction

```python
# Old (v0.x)
from btcpay_greenfield_py import configuration, api_client
config = configuration.Configuration(
    host="https://your-server",
    api_key={"api_key": "YOUR_KEY"},
)
with api_client.ApiClient(config) as client:
    ...

# New (v1.x)
from btcpay_greenfield_py import AuthenticatedClient
client = AuthenticatedClient(
    base_url="https://your-server",
    token="YOUR_KEY",
    prefix="token",  # BTCPay's expected Authorization scheme
)
with client as client:
    ...
```

## Calling endpoints

The old client exposed one **class** per resource (`InvoicesApi`, `StoresApi`,
etc.) with methods. The new client exposes one **module** per endpoint, with
four function variants.

```python
# Old (v0.x)
from btcpay_greenfield_py.api.invoices_api import InvoicesApi
from btcpay_greenfield_py.models.invoice_data import InvoiceData

api = InvoicesApi(client)
invoice = api.invoices_create_invoice(
    store_id="...",
    store_invoice_data=InvoiceData(amount="1.00", currency="USD"),
)

# New (v1.x)
from btcpay_greenfield_py.api.invoices.invoices_create_invoice import sync
from btcpay_greenfield_py.models.create_invoice_request import CreateInvoiceRequest

invoice = sync(
    store_id="...",
    client=client,
    body=CreateInvoiceRequest(amount="1.00", currency="USD"),
)
```

## Sync vs async

The old client was sync-only. The new client provides both:

```python
# Sync
from btcpay_greenfield_py.api.health.health_get_health import sync
health = sync(client=client)

# Async
from btcpay_greenfield_py.api.health.health_get_health import asyncio
health = await asyncio(client=client)
```

Every endpoint module exports four variants:

| Variant | Returns |
|---|---|
| `sync` | Parsed body (or `None` on error) |
| `sync_detailed` | `Response[T]` with status, headers, parsed body |
| `asyncio` | Same as `sync`, but async |
| `asyncio_detailed` | Same as `sync_detailed`, but async |

## Error handling

```python
# Old (v0.x)
from btcpay_greenfield_py.rest import ApiException
try:
    api.invoices_create_invoice(...)
except ApiException as e:
    print(f"HTTP {e.status}: {e.reason}")

# New (v1.x) — use _detailed variants for error inspection
from btcpay_greenfield_py.api.invoices.invoices_create_invoice import sync_detailed
response = sync_detailed(store_id=..., client=client, body=...)
if response.status_code.value != 200:
    print(f"HTTP {response.status_code}: {response.content!r}")
```

The convenience `sync` / `asyncio` variants return `None` on any non-2xx.

## Model field names

Old models used camelCase JSON field names directly. New models use snake_case
Python attributes with `from_dict` / `to_dict` handling the JSON conversion.

```python
# Old
invoice.checkoutLink  # camelCase
invoice.storeId

# New
invoice.checkout_link  # snake_case
invoice.store_id
```

## Authentication

Both versions support the same two auth schemes (API Key + Basic). The key
difference: the new client uses `prefix="token"` for BTCPay's expected
`Authorization: token <key>` header format.

```python
# Old
configuration.api_key["api_key"] = "YOUR_KEY"

# New
AuthenticatedClient(base_url=..., token="YOUR_KEY", prefix="token")
```

## What's gone

- `configuration.Configuration` → replaced by `Client` / `AuthenticatedClient`
- `api_client.ApiClient` → merged into `Client`
- `rest.ApiException` → use `errors.UnexpectedStatus` or check `response.status_code`
- `*.api.*Api` classes → replaced by per-endpoint function modules
- Synchronous-only constraint → both sync + async now
- urllib3 dependency → replaced by httpx
